Royals Player Insights Dashboard

Sports analytics engine with forecasting and causal inference

Role Solo builder (data PM + analyst)
Year 2025
Status Complete
Stack Python · pandas · statsmodels · DoWhy · pybaseball

Problem

MLB player performance data is public but scattered. Standard tools give you box scores and basic stats. Nobody gives you a single local tool that does forecasting, feature importance, causal inference, and directionality analysis on a player you care about — without paying for a vendor platform.

The goal: build a real analytics engine for KC Royals players. Something that goes past "what happened" and into "what drives performance" and "what will happen next."

What the engine does

Module Description
Forecasting SARIMAX time-series model for player metrics across a date range. Falls back to EWMA when statsmodels isn't present.
Feature Importance Identifies which inputs drive the target metric. Surfaces the top features for a given player and season range.
Directionality (PDP) Partial dependence plots show how each feature influences the outcome — not just importance rank but direction and shape.
Causal Inference Uses DoWhy or EconML to estimate a simple causal effect. Separates correlation from causation in the performance signal.
Anomaly Detection Flags outlier games or stretches in the data.

Key Decisions

CLI-first

Streamlit dependency removed from core. Anyone can run python app.py --mode cli --player "Bobby Witt Jr." without a UI install. Streamlit is lazily imported as optional.

Synthetic data fallback

Realistic demo data generated procedurally when real data is unavailable. Makes the tool portable and testable offline — CI/sandbox-friendly.

Soft dependency stack

statsmodels, DoWhy, EconML — all optional. The engine degrades gracefully to simpler methods if they're missing. No hard requirements beyond pandas and numpy.

Example CLI

python app.py --mode cli --player "Bobby Witt Jr." --start-year 2023 --end-year 2025
python app.py --run-tests

All outputs — forecasts, feature importance plots, PDP charts — saved to ./outputs/.

Learnings

  • Graceful degradation is underrated. Writing the fallback logic — synthetic data, EWMA, no-Streamlit mode — took extra time but made the tool actually usable in any environment.
  • Causal inference is a differentiator in sports analytics. Most tools show correlation. Showing a causal estimate — even a simple one — is a different conversation.
  • CLI-first is the right default for data tools that need to run in automation or CI. UI is additive, not the foundation.
